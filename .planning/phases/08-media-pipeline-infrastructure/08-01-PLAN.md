Add a new `bunnyStatus` field to the video schema, placed AFTER the `thumbnailUrl` field and BEFORE the `thumbnailImage` field (keeping it in the storage group with other B2/Bunny fields).

Field definition:

```typescript
defineField({
  name: 'bunnyStatus',
  title: 'Pipeline Status',
  type: 'string',
  description: 'Set by the media pipeline Worker. processing = upload received, ready = CDN validated, error = pipeline failure.',
  options: {
    list: [
      {title: 'Processing', value: 'processing'},
      {title: 'Ready', value: 'ready'},
      {title: 'Error', value: 'error'},
    ],
    layout: 'radio',
  },
  group: 'storage',
  readOnly: true,
  hidden: ({document}) => document?.videoSource !== 'b2',
})
```

Key details:
- `readOnly: true` because this field is machine-populated by the Worker, not editor-editable
- `hidden` uses the same pattern as all other B2 storage fields: `document?.videoSource !== 'b2'`
- `group: 'storage'` places it in the "B2/Bunny Storage" collapsible group
- No `initialValue` for bunnyStatus — it starts undefined until the Worker sets it
- No validation rule — the field is optional (videos can exist without pipeline processing)

  <action>
Update the source-aware video check in `checkCompleteness()` to also verify `bunnyStatus` for B2 videos. The existing code already has a `if (schemaType === 'video')` block with an `isB2` branch. Extend that branch to add a second check for bunnyStatus.

Inside the `if (isB2)` block, AFTER the existing cdnUrl check, add a bunnyStatus check:

```typescript
// bunnyStatus check — B2 videos should have pipeline confirmation
if (doc.bunnyStatus === 'ready') {
  completed++
} else {
  missingFields.push('Pipeline Status')
}
```

Update the total calculation. Currently the video branch returns `checks.length + 1`. Change to `checks.length + 2` to account for the two source-aware checks (cdnUrl + bunnyStatus) for B2 videos.

For wistia/null videos, the total stays at `checks.length + 1` (only videoUrl check). So the return needs to be conditional:

```typescript
if (schemaType === 'video') {
  const videoSource = doc.videoSource as string | undefined
  const isB2 = videoSource === 'b2'
  if (isB2) {
    // CDN URL check (existing)
    if (typeof doc.cdnUrl === 'string' && doc.cdnUrl.length > 0) {
      completed++
    } else {
      missingFields.push('CDN URL')
    }
    // Pipeline status check (new)
    if (doc.bunnyStatus === 'ready') {
      completed++
    } else {
      missingFields.push('Pipeline Status')
    }
    return {completed, total: checks.length + 2, missingFields}
  } else {
    if (typeof doc.videoUrl === 'string' && doc.videoUrl.length > 0) {
      completed++
    } else {
      missingFields.push('Video URL')
    }
    return {completed, total: checks.length + 1, missingFields}
  }
}
```

Also update the `GROQ_FILTERS.video` string. In the B2 branch of the GROQ filter, add a bunnyStatus check. The current B2 line is:
```
(videoSource == "b2" && (!defined(cdnUrl) || cdnUrl == ""))
```

Change to:
```
(videoSource == "b2" && (!defined(cdnUrl) || cdnUrl == "" || !defined(bunnyStatus) || bunnyStatus != "ready"))
```

This means a B2 video is "incomplete" if it lacks cdnUrl OR if bunnyStatus is not "ready".
