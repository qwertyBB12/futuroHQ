## Spacing Scale

Declared values (multiples of 4). @sanity/ui uses numeric `space` props (1–5 maps to 4–20px). Match numeric props to these tokens.

| Token | Value | @sanity/ui `space` | Usage |
|-------|-------|-------------------|-------|
| xs | 4px | space={1} | Icon gaps, badge padding |
| sm | 8px | space={2} | Compact element spacing (progress bar label row) |
| md | 16px | space={3} | Default element spacing (card internal padding) |
| lg | 24px | space={4} | Widget section padding, Stack space between type rows |
| xl | 32px | space={5} | Widget outer padding |
| 2xl | 48px | — | Major section breaks (dashboard widget top margin) |
| 3xl | 64px | — | Page-level spacing (not used in this phase) |

Exceptions:
- Progress bar track height: 8px (widget), 4px (document banner) — sourced from existing components
- Reference array item row height: 44px minimum touch target
- `featuredIn` / `featuredContent` field array items follow Sanity's default reference pill height (~36px internal, 44px with padding)

## Typography

All sizes from Sanity's `<Text size={N}>` and `<Heading size={N}>` system. Pixel equivalents based on Sanity's scale (size 0=12px, 1=13px, 2=15px, 3=18px, 4=21px).

| Role | Sanity `size` | Approx px | Font | Weight | Line Height |
|------|---------------|-----------|------|--------|-------------|
| Body / field label | size={1} | 13px | Mulish | 400 | 1.5 |
| UI label / badge | size={1} | 13px | Mulish | 600 | 1.4 |
| Widget heading | size={1} | 13px | Oswald uppercase | 700 | 1.2 |
| Section heading | size={2} | 15px | Oswald uppercase | 500 | 1.2 |

Rules:
- Widget headings use `fontFamily: "'Oswald', sans-serif"` + `textTransform: 'uppercase'` + `letterSpacing: '0.05em'` — matches EnrichmentProgressWidget pattern exactly.
- Body text inside cards uses `<Text size={1} muted>` for secondary content (counts, percentages, field names).
- Do NOT use Heading sizes above size={2} inside dashboard widgets — sourced from existing widget conventions.

## Interaction Contract

### `featuredIn` array field (content types)
- Input type: array of references — Sanity native UI renders search-to-add with type filter badges
- Placeholder text: "Search people..." (Sanity default reference input placeholder)
- Empty state (no items added): Sanity shows "No items" with an "Add item" button — Sanity native, no custom copy needed
- Reference type filter dropdown shows all 4 types: Alumni, Person, Ledger Person, Collaborator

### `featuredContent` array field (people types)
- Input type: array of references — same Sanity native UI
- Placeholder: Sanity default
- On alumni: replaces `featuredEssays` + `featuredVideos` — old fields will be absent from form after migration

### Alumni featuredEssays/featuredVideos migration
- Mechanism: patch script (not a Studio UI action)
- No dialog or confirmation UI in Studio — script runs in terminal with explicit dry-run/live flag

### Data file format (Claude's discretion — resolved here)
- Format: JSON array (not CSV) — more consistent with Sanity client patch patterns
- Location: `scripts/data/` directory — added to `.gitignore` with comment "personal data files"
- Filename pattern: `{type}-data.json` (e.g. `alumni-data.json`, `collaborators-data.json`, `ledger-persons-data.json`)
- Each record keyed by Sanity `_id` for direct patch targeting
