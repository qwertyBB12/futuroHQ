A document-level form wrapper that renders a completeness banner above the standard form.

   - `Stack`, `Card`, `Flex`, `Text`, `Box` from `'@sanity/ui'`

3. Export the component. The props type depends on the Sanity v5 document.components API. Start with this approach:

```typescript
export function CompletenessInput(props: any) {
  const doc = useFormValue([]) as SanityDocument | null
  const schemaType = doc?._type as string | undefined

  // Guard: skip banner for non-tracked types
  if (!schemaType || !COMPLETENESS_CONFIG[schemaType]) {
    return props.renderDefault(props)
  }

  const {completed, total, missingFields} = checkCompleteness(
    doc as Record<string, unknown>,
    schemaType,
  )
  const pct = total > 0 ? Math.round((completed / total) * 100) : 0
  const isComplete = pct === 100

  return (
    <Stack space={0}>
      <Card padding={3} tone={isComplete ? 'positive' : 'caution'} border style={{borderRadius: 0}}>
        <Stack space={2}>
          <Flex justify="space-between" align="center">
            <Text size={1}>
              {isComplete ? 'Complete' : `${completed}/${total} fields complete (${pct}%)`}
            </Text>
            {!isComplete && (
              <Text size={1} muted>
                Missing: {missingFields.join(', ')}
              </Text>
            )}
          </Flex>
          {!isComplete && (
            <Box style={{background: 'rgba(0,0,0,0.2)', borderRadius: 4, height: 4, overflow: 'hidden'}}>
              <Box style={{width: `${pct}%`, height: '100%', background: COPPER, borderRadius: 4}} />
            </Box>
          )}
        </Stack>
      </Card>
      {props.renderDefault(props)}
    </Stack>
  )
}
```

Key details per UI-SPEC:
- Banner text uses default weight (400 Mulish) — do NOT use `weight="semibold"` or `weight="bold"`
- Progress bar height is 4px (thinner than widget bar per UI-SPEC)
- No transition on banner bar (immediate update per UI-SPEC interaction contract)
- `borderRadius: 0` on Card keeps it flush with form edge
- Complete state: tone="positive", text "Complete", no progress bar rendered
- Incomplete state: tone="caution", text "N/N fields complete (N%)", missing field labels, Copper bar

**Create `scripts/batch-enrich.ts`:**

A CLI batch script that fetches incomplete documents for a specified type and patches them with placeholder/default values. Uses chunked transactions for rate-limit safety.

1. **Imports:**
   ```typescript
   import {config} from 'dotenv'
   import {createClient} from '@sanity/client'
   import {COMPLETENESS_CONFIG, GROQ_FILTERS} from '../lib/completeness'
   ```
   Note: lib/completeness.ts is pure TypeScript with no Studio imports, so it can be imported from Node.js scripts.

2. **Client setup** (matching backfill-governance.ts pattern):
   ```typescript
   config({path: '.env.local', override: false})
   const client = createClient({
     projectId: process.env.SANITY_PROJECT_ID || 'fo6n8ceo',
     dataset: process.env.SANITY_DATASET || 'production',
     token: process.env.SANITY_WRITE_TOKEN || '',
     apiVersion: '2025-01-01',
     useCdn: false,
   })
   ```

3. **`sleep(ms: number)`** — returns a Promise that resolves after ms milliseconds.

4. **`processInChunks<T>(items: T[], chunkSize: number, handler: (chunk: T[]) => Promise<void>, delayMs?: number)`** — iterates items in chunks of chunkSize, calls handler per chunk, sleeps delayMs (default 1000) between chunks. Does not sleep after the last chunk.

5. **CLI argument parsing:**
   - `process.argv[2]` = type name (one of: alumni, collaborator, ledgerPerson, video, podcastEpisode)
   - `process.argv[3]` = optional `--dry-run` flag
   - Validate type name against Object.keys(COMPLETENESS_CONFIG). If invalid, print usage and exit(1).

6. **`run()` function:**
   - Fetch incomplete documents: `client.fetch(\`*[${GROQ_FILTERS[type]}]{_id, _type, ...}\`)` — fetch _id and the fields listed in COMPLETENESS_CONFIG[type] (by field name).
   - Console: `Found ${docs.length} ${type} needing enrichment`
   - If `--dry-run`: log each document's _id and missing fields, then exit.
   - If no docs found: log "All ${type} records are complete." and exit.
   - Process in chunks of 25 using `processInChunks`:
     - Create a transaction: `const tx = client.transaction()`
     - For each doc in chunk: `tx.patch(doc._id, (p) => p.setIfMissing({...defaults}))` where defaults are type-appropriate placeholder values:
       - alumni: `{bio: '[Bio needed]', cohortYear: 2025, generation: '[Generation needed]'}`
       - collaborator: `{bio: '[Bio needed]', orgType: '[Type needed]'}`
       - ledgerPerson: `{openingPortrait: '[Portrait text needed]', currentTitle: '[Title needed]', organization: '[Organization needed]'}`
       - video: `{description: '[Description needed]'}`
       - podcastEpisode: `{description: '[Description needed]'}`
     - Commit transaction: `await tx.commit()`
     - Console: `Patched ${chunk.length} documents`
   - Console: `Done.`

7. **Error handling:** wrap run() call in `.catch(err => { console.error(err); process.exit(1) })`

8. **Usage line** (printed on invalid args): `Usage: npx tsx scripts/batch-enrich.ts <type> [--dry-run]`

The script uses `setIfMissing` so it never overwrites existing values — safe to run multiple times. The placeholder values are intentionally obvious (bracketed text) so they show up in the "Needs Enrichment" filtered lists and can be replaced with real data during Phase 6 data entry.
