## Spacing Scale

Declared values (must be multiples of 4). Applied via `@sanity/ui` `space` props and inline style objects matching the existing glassPanel/glassCard pattern.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon dot diameter, badge letter-spacing |
| sm | 8px | `@sanity/ui` `space={2}` — stack gap between label + count, badge stacking |
| md | 16px | `@sanity/ui` `space={4}` — standard Stack spacing between widget sections; glassCard internal padding (14-16px) |
| lg | 24px | glassPanel internal padding (`padding: 24`) — confirmed in EcosystemHealthWidget |
| xl | 32px | Dashboard grid gap between widget rows |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

**Source:** `components/dashboard/EcosystemHealthWidget.tsx` (padding: 24 on glassPanel, padding: 14-16 on glassCard), `components/dashboard/glassStyles.ts` (borderRadius: 24 on panel, 16 on card).

Exceptions: none. All sizing values on-grid.

Notes:
- Progress bar track height: 8px (widget) / 4px (document banner) — both on-grid. Widget bar is taller for dashboard prominence; banner bar is thinner for inline-form context.
- Entity color dot in widget: 8px diameter — on-grid, consistent with the dot pattern used in EcosystemHealthWidget.

## Typography

All type uses `@sanity/ui` `<Text>` and `<Heading>` components or inline `style` overrides for brand font enforcement. Matches EcosystemHealthWidget exactly.

| Role | Size | Weight | Line Height | Font | Notes |
|------|------|--------|-------------|------|-------|
| Widget heading | 14px (Heading size={1}) | 700 (Oswald) | 1.2 | Oswald | UPPERCASE + letter-spacing: 0.05em — matches EcosystemHealthWidget |
| Widget stat / count | 28px (inline span) | 700 (Oswald) | 1.1 | Oswald | All counts — total and per-type — rendered at 28px |
| Row label | 12px (Text size={1}) | 700 (Oswald) | 1.3 | Oswald | UPPERCASE + letter-spacing: 0.03em — uppercase treatment provides visual distinction from 28px counts |
| Body / muted | 12px (Text size={1}) | 400 (Mulish) | 1.5 | Mulish | Muted counts (e.g. "23/28"), missing field list in banner, "Complete" banner text |

Declared sizes: 12px, 14px, 28px (3 sizes — within maximum of 4).
Declared weights: 400 (Mulish body) and 700 (Oswald headings, counts, and row labels — 2 weights maximum).

**Weight enforcement note:** The banner "Complete" state text and all banner copy must use weight 400 (Mulish body). The RESEARCH.md implementation example uses `weight="semibold"` (600) in a banner component reference — this is NOT within the two-weight contract. Use `weight="regular"` (400) for all banner text elements. The 700 weight is reserved for Oswald headings, widget counts, and row labels only.

**Source:** `components/dashboard/EcosystemHealthWidget.tsx` (font sizes, weight patterns confirmed in JSX), `styles.css` (font-family assignments), `tailwind.config.js` (fontFamily tokens).

## Color

All color values from `CLAUDE.md` palette and confirmed in `tailwind.config.js`.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0E0E0E` (Founder's Black) | Studio body background — set by Sanity theme + styles.css |
| Secondary (30%) | `#1A1714` (Surface Raised) + glassPanel/glassCard surfaces | Widget panels, document pane background, desk sidebar |
| Accent (10%) | `#B17E68` (Copper) | Progress bar fill (both widget bars and document banner bar) ONLY |
| Complete state | `#3D9970` (Green / @sanity/ui `tone="positive"`) | Banner background when 100% complete — not a brand color, uses @sanity/ui semantic tone |
| Destructive | `#813531` (Vermillion Deep) | No destructive actions in this phase — not applicable |

**Source:** CONTEXT.md decision — "Copper (#B17E68) fill color on dark background — consistent with Civic Modern palette." Confirmed in `EcosystemHealthWidget.tsx` (`accent: '#B17E68'` for Hector and NeXT entities).

Accent reserved for:
- Progress bar fill in `EnrichmentProgressWidget` (one bar per tracked type row)
- Progress bar fill in `CompletenessInput` document banner (incomplete state only)

Accent is NOT used for:
- Widget headings (use Sandstone / @sanity/ui default text)
- Row labels (use @sanity/ui default muted text)
- Border or background surfaces (use glassPanel/glassCard)
- Desk list items (use existing Vermillion selected state from styles.css)

Banner incomplete state: `@sanity/ui` `<Card tone="caution">` — uses @sanity/ui's built-in caution yellow-amber tint on dark background. Copper bar overlaid on top.

Banner complete state: `@sanity/ui` `<Card tone="positive">` — green tint background, no progress bar rendered (replaced by "Complete" label).

## Component Inventory

New components this phase (executor reference):

### EnrichmentProgressWidget
- **Location:** `components/dashboard/EnrichmentProgressWidget.tsx`
- **Container:** `div` with `{...glassPanel, padding: 24}` — matches EcosystemHealthWidget exactly
- **Heading:** `<Heading size={1}>` with Oswald, UPPERCASE, letterSpacing 0.05em — text: "Enrichment Progress"
- **Per-type row:** `div` with `{...glassCard, padding: 14}` containing:
  - `<Flex justify="space-between">`: type label (Oswald, UPPERCASE, weight 700, size={1}, letterSpacing 0.03em) + count text (muted, size={1}, format: "23/28 (82%)")
  - Progress bar track: `<Box>` with `background: #1A1714`, `borderRadius: 4`, `height: 8`, `overflow: hidden`
  - Progress bar fill: `<Box>` with `width: pct%`, `background: #B17E68`, `borderRadius: 4`, `transition: 'width 0.4s ease'`
- **Spacing:** `<Stack space={4}>` for heading + rows section; `<Stack space={3}>` between type rows; `<Stack space={2}>` inside each row
- **Data fetch:** `useClient({apiVersion: '2024-10-23'})` + `useEffect` once on mount — no listener
- **Loading state:** Render empty rows with 0% bars until data arrives (no spinner — matches EcosystemHealthWidget pattern of rendering with initial 0 state)
- **Dashboard placement:** New row below the existing EcosystemHealthWidget + SeoHealthWidget row in `DashboardLayout.tsx`

### CompletenessInput (Document Banner)
- **Location:** `components/inputs/CompletenessInput.tsx`
- **Mount point:** Above the document form, injected via `document.components` in `sanity.config.ts`
- **Incomplete state:** `<Card padding={3} tone="caution" border style={{borderRadius: 0}}>` containing:
  - `<Flex justify="space-between">`:
    - Left: `<Text size={1}>` with Mulish weight 400 — format: "3/5 fields complete (60%)"
    - Right: `<Text size={1} muted>` with Mulish weight 400 — format: "Missing: Bio, Photo, Cohort Year"
  - Progress bar track: `<Box>` with `background: rgba(0,0,0,0.2)`, `borderRadius: 4`, `height: 4`
  - Progress bar fill: `<Box>` with `width: pct%`, `background: #B17E68`, `borderRadius: 4`
- **Complete state:** `<Card padding={3} tone="positive" border style={{borderRadius: 0}}>` containing:
  - `<Text size={1}>` with Mulish weight 400 — text: "Complete" (no progress bar rendered)
- **Spacing:** `<Stack space={2}>` inside banner card; `borderRadius: 0` keeps banner flush with form edge (no gap above form fields)
- **Live update:** `useFormValue([])` drives recalculation on every render — no debounce, immediate
- **Guard:** Renders `null` (falls through to `props.renderDefault(props)`) for document types not in `COMPLETENESS_CONFIG`

### lib/completeness.ts (not a UI component, but drives all UI)
- **No visual output** — pure TypeScript, no @sanity/ui imports
- **Exports:** `COMPLETENESS_CONFIG`, `checkCompleteness(doc, type)`, `ENRICHMENT_TYPES` Set
- **Used by:** EnrichmentProgressWidget (GROQ filter strings), CompletenessInput (live validation), deskStructure.ts (GROQ filter), batch-enrich.ts (Node.js validation)

### Desk "Needs Enrichment" List Item
- **Visual treatment:** Standard Sanity desk list item — no custom styling beyond existing styles.css PaneItem rules
- **Label:** Static "Needs Enrichment" — no count in title
- **Icon:** Default document list icon (no custom icon — keeps desk clean)
- **Behavior:** Clicking navigates to filtered document list; documents open with same view tabs as "All [Type]"

## Interaction Contracts

### Progress Bar Animation
- Widget bars: `transition: 'width 0.4s ease'` on fill element — animates from 0% to actual value on first render when data arrives
- Banner bar: No transition (immediate — recalculates as user types, animation would feel laggy)

### Banner Tone Shift (Incomplete → Complete)
- When the last missing field is filled: banner card `tone` shifts from `"caution"` to `"positive"` — @sanity/ui handles the color transition via its internal CSS
- No explicit CSS transition on tone change — the state update causes a re-render, which is instantaneous

### Focus / Keyboard
- Banner is read-only UI — no interactive elements in the banner itself
- First interactive element remains the first form field (natural tab order preserved)
- Banner uses `aria-live="polite"` is not required — @sanity/ui Card rendering handles accessibility; the form's own ARIA structure is unchanged

### Desk Navigation
- "All [Type]" and "Needs Enrichment" are sibling list items under the type's parent list item
- Selecting "Needs Enrichment" shows only published incomplete documents (drafts excluded)
- Documents in the "Needs Enrichment" list open with the same views as "All [Type]" — no behavioral difference once inside the document
