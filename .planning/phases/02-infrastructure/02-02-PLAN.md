  <action>
Configure a Sanity webhook in the manage.sanity.io dashboard. This cannot be automated — webhooks are created through the Sanity Manage UI, not via CLI or Studio code. The existing TriggerDeployAction.ts requires NO code changes — it already works correctly and will serve as the manual fallback.

**Step 1: Get Netlify build hook URL**
- Netlify Dashboard > site > Settings > Build and Deploy > Build hooks
- Copy the hook URL (or create one named "Sanity Auto-Deploy" if none exists)
- This is the same URL used in `SANITY_STUDIO_NETLIFY_BUILD_HOOK_URL`

**Step 2: Create the Sanity webhook**
1. Go to https://manage.sanity.io/projects/fo6n8ceo
2. Navigate to **API** tab > **Webhooks**
3. Click **Add webhook** with these settings:
   - **Name:** `Netlify Auto-Deploy`
   - **URL:** Paste your Netlify build hook URL
   - **Dataset:** `production` (IMPORTANT: only production, NOT staging)
   - **Trigger on:** `Create`, `Update`, `Delete`
   - **Filter:** Leave empty (all document types)
   - **Projection:** Leave empty
   - **HTTP method:** `POST`
   - **HTTP Headers:** None needed
   - **Secret:** Leave empty (Netlify hooks don't need auth)
   - **Status:** Enabled
4. Click **Save**

**Step 3: Verify automated webhook fires**
1. Open Studio (`npm run dev`), use production workspace
2. Open any document, make a trivial edit, publish
3. Check Netlify dashboard — new deploy should appear within ~30 seconds
4. Or check webhook logs: Sanity Manage > API > Webhooks > click webhook > Attempts tab

**Step 4: Verify manual TriggerDeployAction still works**
1. In Studio, open any governed document (e.g., an essay)
2. Click actions menu (chevron next to Publish)
3. Find "Deploy to Sites" action and click it
4. Verify it triggers a deploy in Netlify
5. Both webhook-triggered and manual deploys should appear

**Step 5: Verify staging isolation**
1. Switch to staging workspace
2. Create or edit a document, publish it
3. Confirm NO new deploy appears in Netlify (webhook is scoped to production only)
