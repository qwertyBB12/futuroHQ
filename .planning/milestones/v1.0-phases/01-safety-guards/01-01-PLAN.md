<task type="auto">
  <name>Task 2: Add error feedback to GenerateAIDerivativesAction (SAFE-02)</name>
  <files>components/actions/GenerateAIDerivativesAction.ts</files>
  <action>
The action already correctly uses `SANITY_STUDIO_AI_ENDPOINT` and disables itself with a tooltip when unset. The gap is: errors are silently swallowed (`catch { // Fail silently }`), giving users no feedback when the endpoint is configured but the request fails.

1. Import `useToast` from `sanity` (available in Sanity Studio v3/v5):
   ```typescript
   import {useDocumentOperation, useToast} from 'sanity'
   ```

2. Inside the component, initialize toast:
   ```typescript
   const toast = useToast()
   ```

3. Replace the empty `catch` block with user-facing error feedback:
   ```typescript
   catch (err) {
     toast.push({
       status: 'error',
       title: 'AI generation failed',
       description: err instanceof Error ? err.message : 'Check your AI endpoint configuration',
     })
   }
   ```

4. Add a success toast after the `patch.execute` call:
   ```typescript
   toast.push({
     status: 'success',
     title: 'AI derivatives generated',
   })
   ```

5. Also handle non-ok responses (currently only `if (res.ok)` with no else):
   ```typescript
   if (!res.ok) {
     toast.push({
       status: 'error',
       title: 'AI generation failed',
       description: `Endpoint returned ${res.status}: ${res.statusText}`,
     })
     return
   }
   ```

Keep the existing disabled/tooltip behavior for missing endpoint — that already satisfies the "hidden or visibly disabled" requirement.
